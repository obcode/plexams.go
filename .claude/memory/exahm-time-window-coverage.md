Done 2026-07-08 (branch feat/exahm-time-windows merged to main @ cb9476b). Fixes Issue 2 from the Test26SS-v2 durchspielen: "Embedded Computing" (120 min + 60/60 buffer) was planned 21.07 16:30 into a room booked only 16:00–18:30, and T-Bau rooms showed "free" at 14:30.

**Model** ([[preplanning-seb-exahm]], [[two-phase-exahm-seb]]): an EXaHM exam may only sit where a booked EXaHM room's Anny interval fully covers `[start-pre, start+dur+post]`. Default buffer 30 min each side (`exahmDefaultBuffer`), SHARED 15/15 between two of our own consecutive exams in one room (so a 14:00–18:30 booking fits two 90-min exams at 14:30 & 16:30), full 30 to a booking edge/foreign exam. Overridable per exam via `RoomConstraints.PreExamMinutes/PostExamMinutes` — replaces the default (may widen to real 60 for a lab exam, or shorten to 15). SEB is NOT gated (may overflow to R-Bau).

**Core** `plexams/exahm_intervals.go` (NB: NOT `_windows.go` — that suffix = Go GOOS=windows build constraint, file silently ignored!): `exahmRoomBuffers`, `bookedExahmIntervals` (booked T-Bau rooms as merged time intervals w/ EXaHM/SEB caps via anny.MergeRoomBookings), `exahmWindowSeats` (SEAT-accurate: sums seats of booked rooms whose window fully covers the exam) + `exahmWindowCovered` wrapper, `preplanExamDuration`, `intersectSlotSet`. Tested against the user's exact examples.

**Follow-up 4 (2026-07-09, main @ cbccbb9) — CORRECTION of the packing model:** Rooms MAY be shared between exams (blocking a 30-seat room for 10 students is WRONG per user). So the whole-room "one exam per room" packing (packExamsIntoRooms + its solver/validation hard constraints, all added in follow-ups 2/3) was REMOVED. Capacity is again AGGREGATE booked seats per slot; per-exam booking-window coverage stays (exahmWindowSeats via allowedSlots), plus the canceled-booking filter and SEB window checks. R-Bau threshold is now the SUM of R-building SEB seats (`nonAnnySebSeats`, SEB splits across labs), not the largest single lab; R-Bau-eligible SEB drop cost = `preplanSmallSebDrop + seats` so under tight Anny the SMALLER SEB drop first (bigger kept), never displacing must-place exams. On Test26SS-v2 all 27 pre-exams place, validation clean. Lesson: "Überschneidung der Vor-/Nachläufe" meant the exam window exceeding the booking (canceled-booking + window bug), NOT room-sharing conflicts — don't over-model.

**Follow-up 5 (2026-07-09, main @ 902987c) — cross-slot capacity over time:** Room-sharing (aggregate seats) alone missed that a long/OVERRIDE exam's buffer reaches into a neighbouring slot: Embedded (60-min Vorlauf) needs rooms from 09:30 while the 08:30 exams hold them (shared teardown) until 10:15. Added `exahmOccBuffers` (OCCUPANCY buffers: default shared→half=15 so back-to-back slots abut; override→full) and `cumulativeOverloads` (sweep over time; at every instant the EXaHM and total booked seats occupied by all exams whose occ window covers it must stay within the booking then). Hard constraint in solver (fits + SA Propose via `cumFeasible`) and merged human-readable check in validatePreplan. `preplanUnit` fields dur/occPre/occPost. Result on Test26SS-v2: Embedded lands where it truly fits, all must-place placed, only small SEB spill to R-Bau (warnings), gen ~640ms. Distinction: booking-COVERAGE uses FULL buffers (exahmWindowSeats); room-OCCUPANCY-over-time uses shared/half for defaults.
