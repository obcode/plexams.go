Terminplan (exam schedule) generation — design settled 2026-07-02.

STATUS (2026-07-02): IMPLEMENTED — generic `plexams/optimize` SA core; `plexams/examplan` solver (incremental cost ~1M iters/3s; calibrated DefaultWeights Adjacent 2500/SameDay 900/WorstCase 0.05); builder `buildExamPlanProblem` + `GenerateExamSchedule` (reporter/streaming, writes non-locked entries, gated); GraphQL subscription `generateExamSchedule` + queries `examScheduleConstraints`/`examScheduleConflicts` + conflict-rating CRUD (`setConflictRating`/`setExamsCanShareSlot`/`canShareSlotSuggestions`); planning-state `examScheduleGenerated` + EXAMS gate. CONFLICT MODEL SIMPLIFIED (2026-07): conflicts are per-student only. Pair-level ratings (UNDESIRED/FORBIDDEN) and the "separated"/different-day hard constraint were REMOVED. Kept: per-student acceptance (acceptStudentConflict/removeStudentConflictAcceptance/studentConflictAcceptances; drops that student's proximity penalty, same-slot stays hard) and canShareSlot (pair may run simultaneously; parallel-section suggestions via canShareSlotSuggestions, excluding sameSlot pairs). examScheduleConflicts / examReport.conflicts list only SAME_SLOT/ADJACENT/SAME_DAY (NEXT_DAY dropped, always acceptable); each conflict carries groups1/2, affectedStudents{mtknr,name,program,group,accepted}, infoOnly (both external). Across-day spread uses REAL hours (weekend = cheap). Generation deterministic (seed in examReport.seed). generateExamSchedule.ignoreRatings flag.

EXAM-GAP / travel buffer (done 2026-07): hard constraint `exam-gap` (was "nta-overrun") — a student needs a travel buffer between two of their exams; if an exam's duration (NTA-extended per that student, `duration*(100+delta)/100`) plus buffer reaches into the next slot, that slot is forbidden for them. Buffer is GUI-configurable: SemesterConfigInput.examGapMinutes (+ derived SemesterConfig.examGapMinutes), edited via setSemesterConfigInput/createSemester; effective value = configured else default 30 (legacy planer.examGapMinutes read only on YAML→DB migration). Live: gap 10/30/60 → 62/314/2045 blocks, all feasible. With uniform 2h slots: a 90-min exam is still fine back-to-back, longer/NTA-extended blocks the next slot. examplan Problem tracks nextSlot/prevSlot + SetNTAOverruns([][2]int); feasible()/canSwap() enforce (swap-aware, no cost impact); builder computes forbidden ordered pairs per student; registered for introspection; run reports #pairs. Verified live (dry) Test26SS: buffer 30 → 314 blocks, 106 placed, 0 unplaced, 0 hard (NTA-only was 9).

WARM-START (done 2026-07): generateExamSchedule(keepAssigned) — starts from the exams' current plan (Unit.StartSlot / constructWarm) and does a strict-improvement local search (optimize.Options.StrictImprove: accept only delta<0, no lateral drift), so a re-run only improves and stays close to the previous plan. Off by default (cold generation unchanged; deterministic cold+same-seed still gives exact reproduction). Live Test26SS: warm churn ~24-29 vs cold ~96 after a change.

SHARED CORE (2026-07): DONE — all three solvers share optimize.Anneal. ③ pre-plan PORTED (preplanModel in preplan_solve.go; commit 6894c9b). ④ invigilation PORTED (invigModel in invigplan/optimizer.go; commit c873d86) — deleted its duplicate temperature+loop; output-preserving via new optimize.Options.Rng (the greedy start and the anneal share one rng stream, so the schedule/accept/early-stop and rng are identical to the old inline loop). The progress bridge: invigModel.Converged() caches balance+unfilled (engine calls it when best is snapshotted) → OnProgress reconstructs invigplan.Progress{Balance,Unfilled}. Verified deterministic + converges + all invigplan tests pass. optimize.Options.Rng: use caller rng instead of seeding (for construct+anneal sharing a stream).

INTERIOR-HOLE term (done 2026-07-05): new soft constraint `slot-hole` (examplan) — an empty slot with occupied slots BOTH before and after it on the same day is penalized (Weights.Hole, DefaultWeights 1500 — below Adjacent 2500 so it never creates a directly-consecutive pair to close a hole, above SameDay 900 so it may accept a mild same-day proximity; roomPhase/Phase A sets it 0). Purpose: a free slot mid-day is bad for Aufsichtenplanung; drive it to the day edge, else fill it. ONE term subsumes both remedies the user asked for AND handles the SEB-fixed-in-last-slot case for free (last occupied by fixed SEB ⇒ an earlier empty slot is a hole ⇒ pushed to the first slot). Deliberately BELOW Adjacent(2500)/SameSlot: it only moves/fills when cheap, never creates a student conflict to close a hole (user is fine keeping a hole if fixing it would hurt). Impl: Problem.days/dayOfSlot (slots grouped per exam day, sorted by slot no.); State.holeTotal maintained incrementally in moveUnit (touched days = dayOfSlot[old]/[new]) + Snapshot/Restore; holeCost full-recompute + holeC registered (tier 31); Diagnostics.InteriorHoles reported in the generation log ("freie Slots mitten am Tag: N"). Warm-start (StrictImprove) now also closes holes when strictly improving. Tests: TestHoleDrivesEmptySlotToDayEdge / TestHoleFillsInteriorGap / TestHoleIncrementalMatchesFull. Original agreed design below.

**Architecture:** extract a GENERIC optimization core `plexams/optimize` (simulated annealing: geometric cooling, Metropolis, hard=move-veto/soft=weighted Registry, incremental cost deltas) used by all three solvers. Build order: ① core → ② Terminplan in `plexams/examplan` (the feature) → ③ port the pre-plan solver onto it (its twin) → ④ port the invigilation optimizer (`plexams/invigplan`) → ⑤ GraphQL/GUI/planning-state. The pre-plan solver ([[preplanning-seb-exahm]]) is essentially "Terminplan for a subset" and is the closest analog.

**Unit to schedule:** AssembledExam that is toPlan, not Locked, not external/notPlannedByMe. Domain = AllowedSlots(ancode). Fixed obstacles (untouched, but count for conflicts): Locked plan entries, external + notPlannedByMe (times set via GUI).

**Hard constraints:** fixed placements untouched · SameSlot groups together · no student two exams in same slot (baseline hard, lifted ONLY by pair canShareSlot) · EXaHM per slot ≤ booked EXaHM Anny seats · global seats per slot ≤ total room capacity · allowed slots per exam · "unzulässig"-rated pairs hard-separated else infeasibility report.

**Soft objective (tiers high→low):** ① spread (primary): distance = real time between slot Starttimes (weekends auto-farther); per (student, pair) decreasing penalty; **sum + convex worst-case-per-student**; ladder same-slot(hard)/adjacent-same-day(very high)/same-day(high)/consecutive-days(medium, then ~1/Δ); repeater discount via IsRepeaterExam flag OR group-vs-Groups semester heuristic. ② unerwünscht pairs boosted. ③ section clustering: same module+program, different examer → pull together (canShareSlot removes their conflict; auto-suggest+confirm like [[mucdai-import-linking]]). ④ small-exam (≤5 regs) same-examer → same slot. ⑤ slot-load: convex penalty on total seats/rooms per slot (two 100-exams never together). ⑥ SEB Anny preference (overflow to R allowed).

**Conflict rating (DB, replaces YAML knownConflicts; GUI loop):** two acceptance mechanisms of DIFFERENT strength — pair-level `canShareSlot` (structural, on ZPA/external exam constraints, mirrors pre-plan; removes pair from conflict graph incl. lifting the hard same-slot ban) vs. per-student acceptance `(mtknr, A, B)` (zeroes only that student's SOFT proximity term; same-slot stays HARD for him). Pair ratings 3-level: akzeptiert/unerwünscht/unzulässig (+default). Each regeneration recomputes actual conflicts → show diff (gone/stayed/new) + flag stored ratings "no longer relevant"; option to regenerate ignoring ratings (kept stored).

**Defaults:** warm start both from-scratch and keepAssigned; seed=1 deterministic; adjacency only within a day; NTA time-overrun makes the next slot a hard conflict for affected students; weights calibrated so spread dominates, clustering is a tie-breaker.

**Constraint introspection (read-only):** constraints in the generic core are self-describing (Name/Title/Description + kind hard/soft + weight/tier); a read-only GraphQL query lists, per generator (Terminplan/Pre-Plan/Aufsichten), the currently-applied hard & soft constraints so the user can see what is considered. No configuration via GUI, only understanding.

**Surface:** streaming subscription `generateExamSchedule(dryRun, seed, iterations)` (like AssignInvigilations); writes only non-locked PlanEntries; validate via existing ValidateConflicts; new phase-1 planning-state point "Terminplan generiert" + gate (see [[planning-state-model]]). Reuse: ValidateConflicts already does the per-student same-slot/adjacent/same-day pairwise detection.
